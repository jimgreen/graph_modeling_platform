# 实时画布 DOM 精简设计

日期：2026-07-11

## 目标

在不改变线路选中、右键菜单、双击增加折点、拖拽、重连、图层状态和大模型 LOD 行为的前提下，精简浏览器中实时画布生成的 HTML/SVG DOM。此次范围只覆盖线路及背景页线路，不扩展到设备、文本和量测 DOM。

## 方案比较

### 方案 A：全部压平成单个 `path`

DOM 最少，但会失去 18px 宽命中区，线路难以点击；选中手柄、母线连接点和拖拽状态也没有稳定的共同状态容器。拒绝采用。

### 方案 B：只删除无引用属性

风险最低，但当前实时线路本身没有导出 SVG 中的 `source-layer-id`、`target-layer-id`，结构体积下降有限。仅作为基础清理，不足以达到目标。

### 方案 C：按交互需求分层精简（采用）

编辑线路保留必要的 `connection-group`、宽命中路径和可见路径；只读、背景页和 LOD 路径尽量压平。共同状态只保留一份，重复事件和重复选中线路副本予以删除。

## DOM 规则

1. 普通可编辑线路保留一个 `connection-group`。该容器继续承载线路颜色、选中态、非活动图层状态和唯一的 `data-edge-id`。
2. `connection-hitline` 保留 18px 透明命中宽度，并独占线路的指针、双击和右键事件。
3. `connection-line` 只负责视觉呈现，不再重复绑定与 `connection-hitline` 相同的三个事件；在详细线路组内禁用其指针事件，让事件稳定落到命中路径。
4. 当选中线路的顶层编辑组已经显示时，基础线路层不再重复渲染同一条线路。拖拽或预览状态抑制顶层组时，基础线路仍按现有逻辑显示。
5. 背景页中仅包含单个可见路径的逐线路 `g` 删除，颜色样式和 `background-page-edge` class 直接放到 `path`；背景页总图层组保留。
6. LOD 线路继续保持单个 `path`。`data-edge-id` 保留，因为单节点拖拽的命令式查询依赖它；不增加首末图层或首末设备冗余属性。
7. 线路预览、拖拽影子和变换预览已经是直接 `path`，保持现有扁平结构。

## CSS 与事件

- 将悬停、选中、非活动图层和拖拽原图样式继续建立在 `connection-group` 上，避免把同一状态复制到多个子元素。
- 添加详细线路组内可见路径的 `pointer-events: none` 规则，命中路径仍使用 `pointer-events: stroke`。
- 背景页 CSS 从“容器或容器内线路”双选择器收敛为直接路径选择器。
- 画布空白双击和右键目标判断继续识别 `connection-group`，不改变交互边界。

## 数据与兼容性

- 不修改项目 JSON、后端存储、SVG 导出格式或线路数据模型。
- 不删除实时 DOM 中仍被命令式拖拽查询使用的 `data-edge-id`。
- 不把导出 SVG 的 `source-dev-id`、`target-dev-id` 加入实时编辑 DOM，因为实时事件闭包已经持有线路对象，增加这些属性只会重复数据。

## 验证

1. 自动测试确认背景页线路不再产生逐线路单子元素 `g`。
2. 自动测试确认详细线路的事件只绑定在 `connection-hitline`，视觉路径不重复绑定。
3. 自动测试确认顶层选中线路显示时基础线路不重复输出。
4. 现有模型、线路路由、拖拽和 SVG 导出回归全部通过。
5. 浏览器验证线路单击选择、右键菜单、双击折点、端点重连、空白双击和大模型 LOD 页面无错误。

## 非目标

- 本次不压平设备图元内部为样式、变换或状态切换所需的分组。
- 本次不重构量测编辑、设备标签或 React Flow 预览 DOM。
- 本次不为了减少一个容器而复制更多 `data-*`、class 或内联样式到每个编辑手柄。
