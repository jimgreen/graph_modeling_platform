# 从 SVG 解析生成模型设计

日期：2026-07-12

## 目标

在模型库的方案右键菜单中增加“从 SVG 生成模型”，同时支持两类输入：

1. 对本平台导出的 SVG 进行语义解析，尽可能完整恢复画布、图层、设备、端子节点号、连接线、设备名称、量测和静态图元。
2. 对普通 SVG 执行安全清理后，将完整 SVG 作为一个可移动、缩放、旋转和再次导出的静态图元导入。

生成的模型必须继续使用现有 `ProjectFile`、模型同名冲突处理、后台保存和模型加载流程，不新增平行的模型存储格式。

## 已确认范围

- 入口只放在方案右键菜单中。
- 菜单项名称为“从 SVG 生成模型”。
- 导入目标是被右键点击的方案。
- 平台 SVG 使用完整语义恢复。
- 普通 SVG 整体生成一个静态图元，不拆分内部路径、文字或图形。
- 平台 SVG 中无法识别或缺少设备定义的内容使用静态 SVG 图元回退，优先保证可见内容不丢失。
- 模型名称默认取 SVG 文件名去掉扩展名后的结果。
- 同名模型继续使用现有覆盖、重命名、取消冲突处理。
- 导入成功后必须弹窗提示解析方式、恢复数量和警告数量。

## 方案比较

### 方案 A：语义解析加静态回退（采用）

根据平台 SVG 已有的语义层和属性恢复模型；无法语义恢复的内容作为静态 SVG 图元保留。该方案能够处理当前已经导出的旧 SVG，不需要增加未来 SVG 的体积，符合现有 SVG 精简方向。

### 方案 B：在 SVG 中嵌入完整模型 JSON

未来导入精度最高，但会增大 SVG 文件并重复保存大量模型数据，与现有精简 SVG 体积的要求冲突，不采用。

### 方案 C：根据渲染结果反向识别设备

可以尝试处理更多第三方 SVG，但设备类型、端子、量测和拓扑容易误判，不适合作为电力模型导入主路径，不采用。

## 模块边界

新增独立解析模块 `src/svgModelImport.ts`。该模块只负责把 SVG 文本转换为标准导入结果，不直接操作 React 状态、文件输入框或后台接口。

建议公开以下边界：

```ts
type SvgModelImportMode = "platform" | "generic";

type SvgModelImportStats = {
  nodes: number;
  edges: number;
  measurementGroups: number;
  staticNodes: number;
};

type SvgModelImportResult = {
  mode: SvgModelImportMode;
  project: ProjectFile;
  stats: SvgModelImportStats;
  warnings: string[];
};

parseSvgModel(source: string, options: SvgModelImportOptions): Promise<SvgModelImportResult>;
```

解析模块依赖当前设备模板快照，用于恢复已知设备的默认参数、端子定义和图形行为。UI 层负责将返回的 `ProjectFile` 包装为 `SavedProjectRecord` 并提交到目标方案。

生产环境使用浏览器原生 `DOMParser` 和 `XMLSerializer`，不增加新的前端运行时 XML 解析依赖。

## 平台 SVG 识别

只有满足以下条件时才进入平台语义解析：

- 根节点是合法 `<svg>`。
- 存在 `root_g`。
- 存在平台语义层中的至少一个：`Segment_Layer`、`Text_Layer`、`Measurement_Layer`、`Other_Layer`。
- 存在 `dev-kind`、`dev-id`、`source-dev-id` 或 `target-dev-id` 等平台语义属性中的至少一个。

如果存在平台特征但无法恢复任何有效设备或静态图元，则记录警告并自动转入普通 SVG 静态回退，避免生成空模型。

## 平台 SVG 恢复规则

### 画布和背景

- 根 `viewBox` 的宽高恢复为 `canvasWidth`、`canvasHeight`。
- 没有 `viewBox` 时使用根 `width`、`height`。
- 宽高仍然无效时使用平台默认画布尺寸并记录警告。
- `Background_Layer` 的背景矩形恢复 `canvasBackgroundColor`。
- 背景图片恢复 `canvasBackgroundImage` 和显示方式；平台导出的内嵌图片直接保留。

### 图层

- 从 `export-layer-definitions` 中读取 `layer-id`、名称、可见状态和活动状态。
- 缺少图层定义时创建平台默认图层。
- 设备引用不存在的图层时转入默认图层并记录警告。

### 设备和静态图元

- 解析 `root_g` 下、`defs` 外的所有 `use[dev-kind]` 或其他带 `dev-kind` 的设备节点。
- 优先使用 `dev-id`，其次使用元素 `id` 作为模型节点 ID。
- 重复或不合法 ID 通过确定性后缀生成唯一 ID，并将映射用于连接线、名称和量测恢复。
- `dev-kind` 匹配当前设备模板时，使用模板创建节点，再覆盖 SVG 中明确存在的信息。
- `x`、`y`、`width`、`height` 转换为模型中心坐标和基础尺寸。
- 从引用的 `<symbol>` 第一层几何变换中恢复旋转、`scaleX` 和 `scaleY`。
- 恢复 `idx`、`name`、状态属性、`node`、`node-1` 至 `node-N`、`voltage-type-N` 和 `vbase-N`。
- SVG 没有保存的静态计算参数使用当前设备模板默认值，并记录一条汇总警告，不猜测缺失参数。
- 当前设备库不存在 `dev-kind` 时，从对应 `<symbol>` 提取一个独立 SVG，生成 `static-image` 回退节点，保留原 ID、位置、尺寸、图层和名称。
- `Other_Layer` 中不能映射为已知设备的额外图形，按顶层元素整体转换为静态 SVG 图元。

### 连接线

- 从 `Segment_Layer` 读取 `path`，保留线路 ID、`source-dev-id` 和 `target-dev-id`。
- 解析 `M`、`L`、`H`、`V` 路径命令为线路首末点及中间路径点。
- 使用线路首末点与设备端子画布坐标的距离匹配最近端子。
- 匹配距离超过容差或多个端子同距时，仍保留线路几何路径和首末设备关系，但记录端子匹配警告。
- 无法解析的曲线路径保留为静态 SVG 图元，不伪造拓扑连接。

### 设备名称

- 从 `Text_Layer` 按 `dev-id` 关联设备。
- 恢复 `_labelText`、字体、字号、颜色、粗体、斜体、下划线、锚点、方向和相对设备中心的偏移。
- 找不到设备的独立文本作为静态文本图元恢复。

### 量测

- 从 `Measurement_Layer` 的 `g.mg` 读取 `dev`、`term`、图层和位置。
- `term="tN"` 恢复为对应端子层量测位置；没有 `term` 时恢复为设备层。
- 从量测文本中的 `tspan.mv` 读取量测类型 `mt`、关联字段 `mf`、量测值 ID、前缀、后缀和单位。
- 恢复量测组背景、边框、字体、颜色、字号和位置。
- 量测关联设备或端子不存在时跳过该量测组并记录警告，不创建悬空量测。

## 普通 SVG 静态回退

- 使用根 `viewBox` 或 `width`、`height` 创建模型画布。
- 清理后的完整 SVG 编码为内联 SVG 数据，保存到一个 `static-image` 节点。
- 节点位于画布中心，基础尺寸与 SVG 视口一致。
- 节点允许整体移动、缩放和旋转，并能通过现有 SVG 导出再次输出。
- 不把普通 SVG 内部的 `path`、`rect`、`text`、`image` 等拆成多个模型节点。

## SVG 安全清理

导入前必须执行以下处理：

- 删除 `script`、`foreignObject`、`iframe`、`object` 和 `embed`。
- 删除所有 `on*` 事件属性。
- 删除 `javascript:` 和其他可执行 URL。
- 允许安全的 `data:image/*`、平台图片路径及普通 `http`、`https` 图片引用。
- 相对图片路径无法随单个 SVG 文件一起读取时保留引用并记录警告。
- XML 解析失败时终止导入，不尝试用字符串拼接修复损坏 XML。

## 右键菜单和导入流程

1. 在方案右键菜单的“模型新建”“模型导入”附近增加“从 SVG 生成模型”。
2. 新增隐藏文件输入框，仅接受 `.svg` 和 `image/svg+xml`。
3. 点击菜单项时记录目标方案 ID，然后打开文件选择器。
4. 读取 SVG 文本并调用 `parseSvgModel`。
5. 使用文件名生成默认模型名称。
6. 使用 `createSavedProject` 创建记录。
7. 同名模型复用现有 `pendingModelImportConflict`，支持覆盖、重命名和取消。
8. 成功提交时复用 `commitImportedModelRecord`，完成方案展开、后台保存、模型加载和操作日志。
9. 成功后弹窗显示解析方式、设备数、连接线数、量测组数、静态图元数和警告数。
10. 弹窗最多显示前 20 条警告，完整警告写入运行日志。

## 大文件处理

- 文件读取完成后先显示导入中状态，再开始解析。
- 画布、设备、线路、名称、量测和静态内容分阶段处理。
- 大型设备和线路集合按固定批次处理，批次之间主动让出浏览器事件循环。
- ID 映射、模板映射和端子坐标使用 `Map` 缓存，避免重复线性查找。
- 不在 React 渲染过程中执行 SVG 解析。
- 单次导入只在完整 `ProjectFile` 构建成功后更新模型库，避免产生半导入模型。

## 错误和警告

严重错误会终止导入且不创建模型：

- 文件不是有效 SVG。
- XML 解析失败。
- 无法得到有效画布尺寸，且默认尺寸也不可用。
- 解析结果无法生成有效 `ProjectFile`。

以下情况允许继续并产生警告：

- 平台 SVG 缺少部分语义层。
- 设备类型未知并降级为静态图元。
- 设备静态参数未包含在 SVG 中而使用模板默认值。
- 连接线端子无法唯一匹配。
- 名称、量测或图层引用的设备不存在。
- 重复 ID 被规范化。
- 外部相对图片资源可能无法加载。

## 测试

### 解析单元测试

- 平台 SVG 与普通 SVG 检测。
- 画布、背景、图层和活动图层恢复。
- 单端子、双端子和三端子设备恢复。
- 旋转、非等比缩放和状态恢复。
- 线路首末设备、端子和折线路径恢复。
- 名称内容、位置和字体样式恢复。
- 设备层和端子层量测恢复。
- 未知设备及额外内容静态回退。
- 普通 SVG 只产生一个静态图元。
- 危险元素、事件属性和 URL 清理。
- 重复 ID 的确定性规范化。
- 无效 XML 和空 SVG 错误。

### 往返和性能测试

- 对平台模型执行“导出 SVG、重新导入、再次导出”，比较设备数、连接线数、端子节点号、量测数和画布属性。
- 使用合成的 1000 个设备 SVG 验证批次处理、结果数量和可接受的导入时间。

### 浏览器验证

- 右键方案能够看到“从 SVG 生成模型”。
- 文件选择器仅接受 SVG。
- 平台 SVG 导入后自动打开，并成功保存到后台。
- 普通 SVG 导入后显示为一个静态图元。
- 同名模型覆盖、重命名、取消流程正常。
- 成功和失败弹窗内容正确。
- 导入期间页面没有框架错误覆盖层，完成后控制台没有相关错误。

## 验收标准

1. 当前平台导出的 SVG 可以通过方案右键菜单重新生成模型。
2. 示例 `速度.svg` 恢复 5 个设备、6 条连接线、设备名称、端子节点号和量测组。
3. 普通 SVG 始终只生成一个静态图元，并保持原始整体视觉效果。
4. 未知设备或部分损坏的语义内容不会导致整个 SVG 丢失。
5. 导入模型写入正确的目标方案并保存到后台。
6. 同名冲突行为与 JSON 模型导入一致。
7. 危险 SVG 内容不能进入可执行 DOM。
8. 大型 SVG 导入不会在结果构建完成前产生半成品模型。

## 非目标

- 不对普通 SVG 内部图形进行设备识别。
- 不把普通 SVG 拆成多个可编辑静态图元。
- 不根据图形形状猜测电力设备类型或拓扑。
- 不在 SVG 中新增完整模型 JSON 元数据。
- 不修改现有 SVG 导出格式来补充当前没有导出的全部静态计算参数。
- 不自动下载或打包普通 SVG 引用的外部本地文件。
