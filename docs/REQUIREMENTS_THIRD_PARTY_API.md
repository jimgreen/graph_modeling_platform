# 第三方查询接口需求规格（/api）

> 阶段：需求发现（brainstorm）。本文档仅描述**需求**，不含架构决策、API 契约或实现代码。
> 后续：`/sc:design` 出架构与接口契约，`/sc:workflow` 出实现计划。

## 1. 用户目标

为绘图平台（graph_modeling_platform）新增**供第三方系统调用**的只读查询接口，使外部程序能获取：

1. 方案域数据：方案列表、层级关系、模型列表、方案导出包、模型导出包。
2. 图元库域数据：图元分类、各类图元信息、量测定义、图元定义、模板库信息。
3. 运行时态数据：**当前正在打开的模型**信息——含模型内设备清单、当前选中设备、inspector 三个 tab（【基础】【图元树】【图元】）的内容，以及当前模型的截图（PNG）、SVG、E 文件。

## 2. 范围与约束

| 项 | 约束 |
|----|------|
| 路径前缀 | 所有第三方接口以 `/api/` 开头 |
| 端口 | 接口程序与前端页面**共用同一端口**（同源） |
| 调用方 | 第三方程序（非浏览器前端自身） |
| 操作语义 | 本期仅**只读查询**（GET 为主）；写入不在范围 |
| 数据时态 | 静态存储数据（方案/库/配置）+ 运行时态数据（当前打开模型）两类 |

### 2.1 现状基线（已存在能力，可复用）

server/image-server.mjs 已实现并可作为基础：
- `/api/images`、`/api/image-folders` — 图片素材
- `/api/schemes`、`/api/schemes/export`、`/api/schemes/import` — 方案树与导出导入
- `/api/schemes/project` — 模型读写删
- `/api/color-config`、`/api/measurement-config` — 颜色/量测配置
- `/api/device-library` — 设备/图元/模板库配置（`customDeviceTemplates`、`customAttributeLibraries`、`customComponentTypes`、`customGraphTemplateTypes`、`customGraphTemplates`、`deviceDefinitionOverrides`）
- 服务端 SVG 生成（`buildSvgFile`）、E 文件生成（`buildDeviceParameterFile`）、ZIP 打包（`createSchemeArchiveBuffer`）

> 当前 image-server 监听独立端口（默认 5174，可由 `IMAGE_SERVER_PORT` 环境变量覆盖），前端经 Vite（另一端口）托管。"同端口"为**新增需求**，与现状不同。

## 3. 功能需求

### FR-1 方案域

| ID | 需求 |
|----|------|
| FR-1.1 | 获取方案列表（树形：方案 → 子方案 → 模型记录），含名称、更新时间、层级关系 |
| FR-1.2 | 获取方案层级关系（父子方案树结构） |
| FR-1.3 | 获取指定方案下的模型列表 |
| FR-1.4 | 导出方案数据（方案级打包，含其下全部模型与资源） |
| FR-1.5 | 导出模型数据（单模型的完整数据包） |
| FR-1.6 | 导出模型数据包含其 JSON、SVG、E 文件等制品 |

### FR-2 图元库域

| ID | 需求 |
|----|------|
| FR-2.1 | 获取图元分类（图元库分组结构） |
| FR-2.2 | 获取各分类下的图元信息（设备类型/静态图元种类清单与属性） |
| FR-2.3 | 获取量测定义（量测类型、设备量测绑定档案） |
| FR-2.4 | 获取图元定义（设备定义、设备定义覆盖项） |
| FR-2.5 | 获取模板库信息（自定义图元模板、图元模板类型） |
| FR-2.6 | 获取自定义属性库、自定义设备模板信息 |

### FR-3 运行时态（当前打开模型）

| ID | 需求 |
|----|------|
| FR-3.1 | 获取当前正在打开的模型标识（方案/模型定位 + 元信息） |
| FR-3.2 | 获取当前模型包含的设备清单（节点/边，含参数） |
| FR-3.3 | 获取当前模型**当前选中的设备**（选中节点 id、设备信息） |
| FR-3.4 | 获取【基础】tab 内容（选中设备的基础属性/模型参数） |
| FR-3.5 | 获取【图元树】tab 内容（当前模型图元树结构） |
| FR-3.6 | 获取【图元】tab 内容（选中图元的图元级属性；该 tab 内含 `graph`/`device` 两子视图，接口须反映当前子视图及对应内容） |
| FR-3.7 | 获取当前模型截图（PNG） |
| FR-3.8 | 获取当前模型 SVG |
| FR-3.9 | 获取当前模型 E 文件 |

### FR-4 通用

| ID | 需求 |
|----|------|
| FR-4.1 | 接口错误统一格式（error 结构、合适 HTTP 状态码） |
| FR-4.2 | 大响应支持压缩（gzip）与条件请求（ETag/304）——沿用现状能力 |
| FR-4.3 | 运行时态不可用时（如无前端在线）返回明确状态而非崩溃 |

## 4. 非功能需求

| ID | 需求 |
|----|------|
| NFR-1 | 同源同端口：第三方与前端访问同一 host:port，无跨端口；dev/prod 一致 |
| NFR-2 | 只读接口对前端性能无可感知影响（运行时态数据采集不得阻塞画布） |
| NFR-3 | 运行时态时效性：实时秒级（请求即拉取，无缓存延迟） |
| NFR-4 | 接口版本化：第三方查询统一走 `/api/v1/`，支持后续破坏性演进 |
| NFR-5 | PNG 截图为当前画布可视内容，满足第三方识别图元与拓扑需求 |
| NFR-6 | 并发：多个第三方调用可并发读取且互不干扰 |
| NFR-7 | 无鉴权，内网信任（沿用现状） |
| NFR-8 | 预留写入能力（命名/分层留口），本期仅实现只读 |
| NFR-9 | 每个接口必须有测试用例；接口代码覆盖率 100%（行+分支）；`pnpm test:all` 全绿方算开发成功 |
| NFR-10 | 每接口定义 AC 清单（正常 + 404 不存在 + 400 参数非法；运行时态另含无在线客户端状态），全过方算成功 |

## 5. 用户故事与验收标准

### US-1 第三方拉取方案目录
**作为**外部集成系统，**我想**获取方案树与模型清单，**以便**知道有哪些可用的方案和模型。
- AC1: 调用方案列表接口返回完整方案树（含层级与模型记录摘要）
- AC2: 树节点含名称与更新时间
- AC3: 空库时返回空结构而非错误

### US-2 第三方导出方案/模型
**作为**外部系统，**我想**下载方案或模型的完整数据包，**以便**离线分析或归档。
- AC1: 方案导出返回含全部子项的打包制品
- AC2: 模型导出返回单模型 JSON + SVG + E 制品
- AC3: 导出内容与前端"保存后"状态一致

### US-3 第三方查询图元库
**作为**外部系统，**我想**获取图元分类、图元定义、量测定义、模板库，**以便**理解图元语义与可用类型。
- AC1: 返回图元分类树与各分类图元种类
- AC2: 返回量测类型定义与设备量测绑定
- AC3: 返回设备定义（含覆盖项）与模板库

### US-4 第三方感知当前打开模型
**作为**外部系统，**我想**知道前端当前打开了哪个模型、选中了什么、inspector 各 tab 显示什么，**以便**实时感知用户操作上下文。
- AC1: 返回当前打开模型定位信息（无打开模型时明确返回"无活动模型"状态）
- AC2: 返回当前选中设备 id 与设备信息
- AC3: 返回【基础】【图元树】【图元】三 tab 的结构化内容
- AC4: 选中设备/无选中时各 tab 内容相应变化

### US-5 第三方获取当前模型制品
**作为**外部系统，**我想**获取当前模型的 PNG 截图、SVG、E 文件，**以便**复用渲染结果与设备参数。
- AC1: PNG 截图为当前画布可视内容
- AC2: SVG 与前端导出 SVG 一致
- AC3: E 文件含设备参数表（含拓扑节点号）
- AC4: 三种制品可分别独立获取

## 6. 决策记录（已拍板）

| OQ | 议题 | 决策 |
|----|------|------|
| 1 | 同端口部署 | image-server 作唯一入口，dev/prod 同端口托管静态资源 + `/api`；dev 内嵌 Vite middleware 保 HMR |
| 2 | 运行时态桥接 | server 收第三方请求 → 经 WebSocket 向在线前端按需拉取 |
| 2.1 | 多客户端 | 第三方调用时可指定客户端 ID；不指定时取最近活跃客户端（默认策略 design 阶段定） |
| 3 | 时效性 | 实时秒级（请求即拉，无缓存延迟） |
| 4 | PNG 截图 | 前端 canvas 截图后经 WS 回传 server，再返第三方；截图为当前画布可视内容 |
| 5 | 鉴权 | 无鉴权，内网信任（沿用现状 CORS `*`） |
| 6 | 版本化 | 第三方查询统一走 `/api/v1/`；旧内部路径（`/api/images` 等）迁移策略留 design 阶段 |
| 7 | 写入 | 预留写入能力（命名/分层留口），本期仅实现只读 |
| 8 | schema 粒度 | 完整复刻前端 inspector 展示结构（含分组/标签/值），开箱即用 |
| 9a | E 文件 | 前端"导出 E 模型文件"按钮产物（`exportEFile` → `<name>.e`，定义在 `model.ts`），非 server 端 `buildDeviceParameterFile`；design 阶段统一二者 |
| 9b | 导出包格式 | 沿用现有 ZIP（`createSchemeArchiveBuffer`） |

### 派生约束（供 design 阶段落实）
- 运行时态接口要求前端在线；前端不在线时返回明确"无在线客户端/不可用"状态（对应 FR-4.3）。
- server 需维护客户端注册表（WS 连接表 + 最近活跃时间 + 客户端 ID）。
- 前端需实现：(a) WS 客户端连入与注册；(b) 响应 server 拉取请求，序列化三 tab 展示结构；(c) canvas 截图 PNG 回传。
- 旧 `/api/*` 内部路径与新 `/api/v1/*` 第三方路径并存期间需明确边界，避免前端误用第三方接口。
